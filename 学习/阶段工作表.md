# vsinkore 阶段工作表

VSCode 插件：把 `@inkore/editor-core` 作为 host，注册为 `.md` 的 Custom Text Editor。纯免费、只做编辑。

**定位**：面向开发者的文档查阅 / 编辑场景——在 VSCode 里所见即所得地看 / 改项目文档、快速定位信息。**视觉完全对齐 VSCode**（用原生主题色变量与字体、跟随明暗），不移植砚客主题。`[D0-5]`

决策 / 契约 / 踩坑见 [阶段决策备忘.md](阶段决策备忘.md)（`[Dx-y]` 锚点）。

## 阶段二：内核跑起来（PoC，只读不回写）

- [X] webview 引入内核：`createEditorState`（`saveImage` 传 no-op）+ 全套 NodeView wiring（参照 `example/main.ts`）
- [X] 排版 CSS：取桌面版 Editor.module.css 的**排版结构**（行距 / 间距 / 块样式），颜色 / 字体一律改用 VSCode 原生变量，对齐本仓 `原型/editor-mockup.html` `[D0-5]` `[D2-1]`
- [X] extension 读 `document.getText()` → push 给 webview → `parseMarkdown` 渲染
- [X] katex / prosemirror CSS 打包（不走 CDN）
- [X] 代码块语法高亮：Shiki（JS 引擎，无 WASM）+ ProseMirror inline decoration 给 `<code>` 上色（不动 contentDOM，保留内核 header）；按需懒加载语言；nonce 授权 `<style>` 注入 token 色表（严守 CSP）；主题跟随 VSCode 明暗 `[D0-8]`。F5 验证：dark-plus 配色生效、零 CSP/chunk 报错
- [X] 修复：语言下拉被 `.pm-code-block { overflow:hidden }` 裁切。去掉外层 overflow，圆角改由 header（上两角）/ pre（下两角）各自承担。F5 验证下拉完整展开
- [X] 验证：打开真实 `.md` → 看到所见即所得渲染，配色对齐原型，能编辑（先不回写磁盘）

**渲染异常已复现 + 分类**（headless 跑 `parseMarkdown` 定位）：
- _成因一（已修 · F5 验证）_：math / 脚注 / 代码块下拉等 nodeView **缺 host CSS**，渲染裸 DOM。已在 `media/editor.css` 补齐，贴合代码块卡片风格、全 `--vscode-*` 变量。F5 渲染正常。
- _成因二（待办 · 内核侧）_：标准 Markdown 脚注定义 `[^1]: 文本` 内核不解析（只认自身 `<fn-def>` HTML 形式），且往返一趟会丢定义、乱编号。内核源码已内联（见下），可作为独立任务在 `src/editor-core/` 内修——需连带解析 / 序列化 / 同步插件三处一起改，并定序列化口径。

**结构变更：内核源码内联** `[D-inline]`：内核由 `file:../Inkore-core` 改为源码拷入 `src/editor-core/`，依赖收编进本仓，删 esbuild ProseMirror alias（`[D2-2]` 修法作废，单 node_modules 自然去重）。`npm ls` 单副本、构建 + typecheck 通过。
